# Dynamic-Programming-for-River-Water-Allocation-with-Pollution-Control
A fixed supply of water Q must be distributed among N industrial plants located sequentially along a river. Each plant consumes some water and generates waste proportional to its consumption, which flows downstream and increases the pollution level faced by subsequent plants.


he objective is to minimize the total regional treatment cost, which depends on both the water consumed and the upstream waste concentration at each plant.
The cost function at each plant is c_n(q, w) = αₙ × (q × w), meaning cost increases with both the amount of water consumed and the level of upstream waste — plants further downstream face higher costs for the same water use if upstream plants have polluted heavily. Each plant's waste contribution is aₙ × q, which accumulates into the downstream waste level, creating a coupling between successive stages that makes this a genuine multi-stage sequential decision problem.
The state is a two-dimensional pair (s, w): the remaining water supply and the current upstream waste level. Because waste evolves continuously along the river depending on upstream allocations, all reachable (s, w) pairs are first enumerated via a forward pass before backward induction begins, ensuring the DP only evaluates states that can actually be reached from the initial conditions.
The terminal condition requires all water to be consumed — any remaining water at the end is infeasible. Backward induction then solves from the last plant to the first: for each reachable state, every feasible water allocation is evaluated, and the one minimizing immediate cost plus optimal future cost is selected.
The output includes the number of reachable states per stage, a detailed DP table for small instances, the optimal water allocation for each plant, the associated costs and waste progression, and a verification check confirming that the sum of stage costs matches the DP objective value.
